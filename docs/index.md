# QNX Helper

这是一个面向个人学习与长期积累的 QNX 文档站点，重点记录：

- QNX 基础知识梳理
- 开发环境搭建过程
- 常用命令与实践经验
- 调试方法与问题排查
- 学习过程中的踩坑记录

## 站点目标

这个项目不只是“收集资料”，而是尽量把 QNX 学习过程沉淀成可检索、可复用、可持续补充的知识库。

## 适合什么人看

- 刚接触 QNX，想先建立整体概念的人
- 已经开始写程序，但经常卡在环境、部署和调试环节的人
- 做板级开发、驱动调试或系统集成，需要长期沉淀经验的人

## 建议的使用方式

你可以按下面的顺序阅读和维护内容：

1. 先看“入门”部分，建立整体认识。
2. 再看“基础概念”，理解 QNX 的核心机制。
3. 在“开发实践”中积累命令、编译和调试经验。
4. 把实际问题统一记录到“问题记录”里，避免重复踩坑。

## 推荐学习路线

### 第一阶段：先建立系统观

重点理解这些问题：

- QNX 为什么强调微内核
- 为什么消息传递在 QNX 中这么重要
- 进程、线程、服务之间是怎样协作的

这一阶段先不要急着记太多命令，先把核心模型搞清楚。

### 第二阶段：开始能动手

建议开始做几类最小实验：

- 编译并运行一个最小程序
- 查看系统中的进程和线程
- 观察消息传递或服务调用行为
- 练习部署程序到目标环境

### 第三阶段：进入真实问题

当你开始遇到“程序起不来”“线程卡住”“服务连不上”“部署后缺库”这类问题时，学习速度会明显变快。这个阶段最重要的不是记结论，而是把问题的现象、排查过程和根因记下来。

## 建议长期维护的栏目

- 基础概念：写成稳定的主文档，避免重复整理
- 开发实践：记录高频命令、编译方法、调试流程
- 问题记录：每篇只解决一个明确问题
- 参考资料：整理官方文档入口和自己的学习路线

## 后续可继续补充的方向

- 启动流程分析
- BSP 与驱动开发
- 串口、网络、文件系统实验记录
- IPC 示例与源码解读
- 常见错误案例库
