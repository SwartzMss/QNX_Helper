# 学习路线与资料

## 建议学习路线

1. 先理解 QNX 的系统定位和微内核思路
2. 再学习进程、线程和消息传递
3. 然后进入编译、部署和调试实践
4. 最后逐步深入驱动、启动流程和系统组件

## 每个阶段建议产出什么

### 入门阶段

至少形成两类笔记：

- 一篇“QNX 是什么”的总览
- 一篇自己的环境搭建记录

### 基础阶段

建议形成三篇稳定主文档：

- 微内核与消息传递
- 进程与线程
- 常见系统组件或服务模型

### 实践阶段

重点不是写概念，而是积累：

- 常用命令手册
- 编译部署步骤
- 调试案例
- 常见报错记录

### 深入阶段

可以逐步扩展到：

- 启动流程
- BSP
- 驱动开发
- 文件系统
- 网络栈
- 性能分析

## 建议补充的资料分类

- 官方文档
- 开发工具手册
- 板级支持包相关资料
- 社区经验文章
- 自己整理的实验记录

## 个人维护建议

- 一类问题只保留一篇主文档，避免重复
- 命令类内容集中整理，便于检索
- 实验记录尽量写清环境和步骤
- 问题记录优先写“现象”和“根因”

## 推荐的写法

如果是概念页：

- 先讲它解决什么问题
- 再讲它在系统里扮演什么角色
- 最后再写术语和接口

如果是实践页：

- 先写目标
- 再写步骤
- 然后写常见错误

如果是问题记录：

- 先写现象
- 再写排查过程
- 最后写根因和解决方法
