# 常用命令记录

这个页面适合持续追加，用来沉淀开发中最常查的 QNX 命令。

## 进程与线程

```bash
pidin
pidin threads
pidin mem
```

### `pidin`

用途：查看系统中的进程、线程和运行状态，是排查问题时最常用的入口之一。

常见场景：

- 确认某个服务是否已经启动
- 查看线程是否阻塞
- 观察内存、优先级和线程状态

建议优先记住的思路不是参数本身，而是“先用 `pidin` 看活没活，再看线程卡在哪”。

### `pidin threads`

用途：查看线程级别的信息。

常见场景：

- 程序无响应时看线程是否阻塞
- 排查线程优先级问题
- 定位是单个线程异常还是整个进程异常

### `pidin mem`

用途：查看进程相关的内存信息。

常见场景：

- 排查内存占用异常
- 对比不同阶段的资源变化
- 辅助判断是否存在泄漏或异常增长

## 文件与系统信息

```bash
ls
df -h
mount
uname -a
```

### `mount`

用途：查看当前挂载的文件系统。

常见场景：

- 确认某个分区是否已挂载
- 排查文件访问失败是否与挂载状态有关

### `uname -a`

用途：查看系统基本信息。

常见场景：

- 记录当前目标环境信息
- 远程协作时快速确认系统版本

## 网络相关

```bash
ifconfig
netstat -an
ping <ip>
```

### `ifconfig`

用途：查看和配置网络接口。

常见场景：

- 确认网卡是否起来
- 检查 IP 地址是否正确
- 网络不通时先做基础检查

### `netstat -an`

用途：查看网络连接和端口状态。

常见场景：

- 确认服务端口是否监听
- 检查连接是否建立
- 辅助分析通信异常

### `ping <ip>`

用途：测试基础网络连通性。

常见场景：

- 判断目标板与主机是否可达
- 区分是“网络不通”还是“应用服务异常”

## 推荐的排查顺序

遇到问题时可以先按这个顺序看：

1. 用 `pidin` 看进程和线程状态。
2. 用系统信息命令确认环境是否正常。
3. 涉及网络时再看 `ifconfig`、`netstat` 和 `ping`。
4. 把最终有效命令沉淀成固定案例。
