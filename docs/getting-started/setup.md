# 环境搭建

## 目标

搭建一套可以用于 QNX 学习、编译和调试的基础环境。

对于个人学习来说，这一页最重要的价值不是“列安装步骤”，而是把你的环境固定下来。很多后面出现的问题，最终都和版本、工具链、目标板连接方式有关。如果这里记录得清楚，后续排查会轻松很多。

## 建议记录的内容

- 主机系统版本
- QNX SDP 版本
- 安装路径
- 许可证配置方式
- 交叉编译工具链路径
- IDE 或命令行工具使用方式

## 为什么要认真记录环境

QNX 学习过程中经常会碰到下面这些问题：

- 同一份代码在另一台机器编不过
- 工具链命令能执行，但目标架构不对
- 程序编过了却无法在目标环境运行
- 调试链路不通，但表面上看起来都配置过了

这些问题很多都不是“代码逻辑错误”，而是环境信息不完整导致的。因此建议把这页当成你自己的环境基线文档。

## 可继续补充的章节

### 1. 安装前准备

- 需要的系统环境
- 安装包来源
- 许可证准备

### 2. 安装过程

- SDP 安装步骤
- 环境变量配置
- 常见安装报错

### 3. 验证环境

- 编译一个最小示例
- 检查工具链是否可用
- 验证目标板连接与日志输出

## 建议的验收标准

当下面几件事都能完成时，环境通常就算基本可用了：

1. 能正常调用编译工具链。
2. 能生成目标程序。
3. 能把程序放到目标环境。
4. 能在目标环境启动程序并看到结果。
5. 出问题时能拿到基本日志和进程信息。

## 常见问题方向

- 命令存在但环境变量不完整
- 工具链版本和目标系统版本不匹配
- 目标板连接正常但部署路径不对
- 依赖库缺失导致程序无法启动
- 调试工具可连通但符号或目标文件不匹配
